[05/06/2013]

- Amphi de présentation du sujet : principaux thèmes abordés, planning, groupes...
- Première approche des différentes parties du projet :
---- Un fichier TXT est parsé grâce à un parser Ocaml. On crée alors une structure Ocaml correspondante
---- Cette structure est utlisée pour générer un fichier XML
---- Le fichier XML est parsé par une API Java (type DOM ou SAX) qui se charge de créer une structure objet (collection d'objets ?)
---- Cette structure objet est exploitée par le moteur de jeu
- Travail à réaliser en Ocaml :
---- Ecrire une grammaire puis un parser capable d'exploiter un fichier texte standardisé (optionnel dans un premier temps)
---- Créer une structure Ocaml exploitable
---- Générer un fichier XML à partir de cette structure
- Travail à réaliser en Java :
---- Parser un fichier XML et générer une structure objet adéquate (à approfondir...)
---- Développer un moteur de jeu qui applique des règles prédéfinies sur une collection d'objets (de lemmings ?)
---- Développer une interface permettant d'afficher le jeu (console pour debugger puis graphique avec Swing)
- Architecture retenue :
---- Classe : Jeu, Moteur, Carte, Extraction, Interface (Console,Graphique)
- Etablissement d'une première liste de routines à coder dans les différentes classes

- Points à développer (répartis entre les membres du groupe, à débattre demain) : 
----- Passage du fichier XML à une structure Java : classe Automate recquise ?
----- Rédaction d'un fichier XML "type"
----- Rédaction de la grammaire du parser Ocaml
----- Rédaction des automates de trois lemmings "standards"
----- Utlisation d'un logiciel de subversion : SVN
----- Ocaml : type à utiliser et concept de lecture/écriture dans des fichiers
----- Etude rapide de la bibliothèque SWING afin d'estimer son potentiel dans notre projet
